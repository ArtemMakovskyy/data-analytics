## 0. Загальна карта аналізу

П'ять послідовних етапів: кожен наступний крок спирається на висновки попереднього.

```mermaid
flowchart TD
    A["1. Опис розподілу даних<br/><small>діапазон, квартилі, IQR, std</small>"] --> B["2. Виявлення викидів<br/><small>візуально, IQR, Z-score</small>"]
    B --> C["3. Формування вибірки<br/><small>random / stratified / systematic / cluster / bootstrap</small>"]
    C --> D["4. Оцінка надійності<br/><small>довірчий інтервал, p-value, нормальність</small>"]
    D --> E["5. Кореляція<br/><small>Пірсон / Спірмен / Кендалл</small>"]

    style A fill:#E6F0F8,stroke:#2F5D8A
    style B fill:#FBEBE2,stroke:#B5562F
    style C fill:#E2F2EC,stroke:#1E7A63
    style D fill:#ECE8F6,stroke:#5B4B8A
    style E fill:#FBEAF0,stroke:#A23A6B
```

## 1. Виявлення та обробка викидів

```mermaid
flowchart TD
    Start["Є набір даних"] --> Viz["Спочатку візуалізуй<br/>(lineplot / scatterplot)"]
    Viz --> Q{"Видно різкі<br/>піки чи провали?"}

    Q -->|так| IQR["Метод IQR<br/><small>межі: Q1−1.5·IQR / Q3+1.5·IQR</small><br/><small>стійкий до викидів</small>"]
    Q -->|не зрозуміло| Z["Метод Z-score<br/><small>викид, якщо &#124;Z&#124; &gt; 3</small><br/><small>чутливий до викидів</small>"]

    IQR --> Found{"Викид знайдено —<br/>що далі?"}
    Z --> Found

    Found -->|це помилка даних| Remove["Видалити / виправити<br/><small>не впливає на висновки</small>"]
    Found -->|реальна подія<br/>акція, свято| Keep["Залишити й пояснити<br/><small>враховувати у прогнозах</small>"]

    style IQR fill:#FBEBE2,stroke:#B5562F
    style Z fill:#FBEBE2,stroke:#B5562F
    style Remove fill:#F8E5E2,stroke:#A23A2E
    style Keep fill:#EBF4E3,stroke:#3B7A23
```

**Логіка рішень:**
- Якщо аномалія очевидна на графіку → застосовуй **IQR** (стійкіший до екстремальних значень, працює навіть без нормального розподілу).
- Якщо неочевидно → перевір через **Z-score** (краще працює на приблизно нормальних даних, але сам чутливий до викидів, бо рахується через середнє).
- Знайшов викид → з'ясуй причину: технічна помилка → прибрати; реальна подія → залишити й врахувати в прогнозах.

## 2. Методи формування вибірки

```mermaid
flowchart TD
    Start["Велика популяція даних"] --> Q{"Чи є природні групи /<br/>структура в даних?"}

    Q -->|немає| Random["Простий випадковий<br/><small>.sample(n=...)</small>"]
    Q -->|категорії| Strat["Стратифікований<br/><small>.groupby().sample()</small>"]
    Q -->|впорядкований ряд| Sys["Систематичний<br/><small>.iloc[::n]</small>"]
    Q -->|регіони / кластери| Cluster["Кластерний<br/><small>обрати кілька кластерів</small>"]

    Random --> Small{"Вибірка мала або<br/>розподіл невідомий?"}
    Strat --> Small
    Sys --> Small
    Cluster --> Small

    Small -->|ні| Done["Вибірка готова до аналізу<br/><small>переходь до кроку «надійність»</small>"]
    Small -->|так| Bootstrap["Bootstrap-метод<br/><small>повторна вибірка із заміщенням</small>"]

    style Random fill:#E2F2EC,stroke:#1E7A63
    style Strat fill:#E2F2EC,stroke:#1E7A63
    style Sys fill:#E2F2EC,stroke:#1E7A63
    style Cluster fill:#E2F2EC,stroke:#1E7A63
    style Done fill:#EBF4E3,stroke:#3B7A23
    style Bootstrap fill:#ECE8F6,stroke:#5B4B8A
```

**Логіка рішень:**
- Немає структури в даних → **простий випадковий вибір**.
- Є категорії (типи товарів тощо) → **стратифікована вибірка** — по рівній кількості з кожної категорії.
- Дані впорядковані (за часом) → **систематична вибірка** — кожен n-й елемент.
- Дані розкидані по регіонах/групах → **кластерна вибірка** — вибрати кілька цілих кластерів.
- Якщо вибірка мала або розподіл популяції невідомий → **bootstrap**, незалежно від обраного методу семплування вище.

## 3. Нормальність, довірчий інтервал та p-value

```mermaid
flowchart TD
    Start["Маю вибірку даних"] --> Size{"Розмір вибірки<br/>великий чи малий?"}

    Size -->|великий| DAgostino["тест Д'Агостіно-Пірсона<br/><small>normaltest()</small>"]
    Size -->|малий| Shapiro["тест Шапіро-Уілка<br/><small>shapiro()</small>"]

    DAgostino --> P{"p-value тесту<br/>нормальності"}
    Shapiro --> P

    P -->|≥ 0.05| Normal["Розподіл нормальний<br/><small>можна Пірсона, t-тести</small>"]
    P -->|&lt; 0.05| NonNormal["Розподіл ненормальний<br/><small>непараметричні методи, Спірмен</small>"]

    style DAgostino fill:#E6F0F8,stroke:#2F5D8A
    style Shapiro fill:#E6F0F8,stroke:#2F5D8A
    style Normal fill:#EBF4E3,stroke:#3B7A23
    style NonNormal fill:#FAF0DB,stroke:#9A6B16
```

**Логіка рішень:**
- Велика вибірка → **тест Д'Агостіно-Пірсона** (потужніший на великих даних).
- Мала вибірка → **тест Шапіро-Уілка** (надійніший на малих даних).
- p ≥ 0.05 → немає підстав відхиляти гіпотезу про нормальність → можна параметричні методи (Пірсон, t-тести).
- p < 0.05 → розподіл ненормальний → переходь на непараметричні методи (Спірмен, Кендалл).
- Цей самий принцип `p < 0.05` діє для будь-якого тесту значущості далі (кореляція, t-тест, A/B-тест).

## 4. Вибір методу кореляції

```mermaid
flowchart TD
    Start["Потрібно знайти кореляцію"] --> Q1{"Дані кількісні, неперервні,<br/>нормальний розподіл?"}

    Q1 -->|так| Pearson["Пірсон (pearsonr)<br/><small>лінійний зв'язок</small>"]
    Q1 -->|ні| Q2{"Є викиди / аномалії<br/>в ранжуванні?"}

    Q2 -->|ні| Spearman["Спірмен (spearmanr)<br/><small>монотонний / ранговий зв'язок</small>"]
    Q2 -->|так| Kendall["Кендалл (kendalltau)<br/><small>стійкий до аномалій</small>"]

    Pearson --> PCheck["Перевір p-value<br/>тесту значущості"]
    Spearman --> PCheck
    Kendall --> PCheck

    PCheck -->|p ≥ 0.05| NotSig["Зв'язок не значущий<br/><small>ігноруємо величину кореляції</small>"]
    PCheck -->|p &lt; 0.05| Sig["Зв'язок значущий<br/><small>оцінюємо силу та напрямок</small>"]

    style Pearson fill:#E6F0F8,stroke:#2F5D8A
    style Spearman fill:#E2F2EC,stroke:#1E7A63
    style Kendall fill:#ECE8F6,stroke:#5B4B8A
    style Sig fill:#EBF4E3,stroke:#3B7A23
    style NotSig fill:#F1EFE8,stroke:#888888
```

**Логіка рішень:**
- Кількісні неперервні дані з нормальним розподілом → **Пірсон** (лінійний зв'язок, −1…+1).
- Інакше: якщо немає аномалій → **Спірмен** (монотонний/ранговий зв'язок).
- Інакше: якщо є аномалії → **Кендалл** (найстійкіший до викидів).
- Завжди перевіряй **p-value** перед інтерпретацією величини коефіцієнта:
    - `p ≥ 0.05` → результат міг статись випадково, висновки про зв'язок робити не можна.
    - `p < 0.05` → зв'язок статистично значущий, можна оцінювати силу (`|r|` ближче до 1 — сильніший) і напрямок (знак).

## Як користуватись цією картою

1. Почни зі схеми **0** — визнач, на якому етапі аналізу ти зараз.
2. Для відповідного етапу відкрий деталізовану схему (1–4).
3. Рухайся по ромбах-питаннях (умовах) — кожна відповідь веде до конкретного методу або дії.
4. Кінцеві блоки кожної схеми (зелені/кольорові) — це рішення або метод, який треба застосувати.
